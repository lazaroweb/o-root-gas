# GAS App Kit — Guardrails

These rules are always active. Never violate them.

## 1. TypeScript strict mode

- `tsconfig.json` must have `"strict": true`.
- No `any` types. Use `unknown` and narrow it, or create proper interfaces.
- Every component prop and server response must be typed.

## 2. `.clasp.json` rootDir protection

**The `rootDir` in `.clasp.json` must always be `"./dist"`.**

Never set it to `"."`, `"./"`, or omit it. If `clasp create` overwrites the file and drops `rootDir`, add it back immediately. Pushing from the project root uploads `node_modules` and freezes Google Apps Script.

## 3. Bundle size limits

- **Hard limit: 1.5MB** — `dist/App.html` must not exceed 1.5MB. The push script blocks deploys above this limit.
- **Warning threshold: 1.3MB** — warn the user and suggest reducing imports.
- Tips to reduce bundle size:
  - Import Ant Design components individually: `import { Button } from 'antd'` (esbuild tree-shakes automatically)
  - Avoid importing entire libraries when only one function is needed
  - Check bundle with: `ls -lh dist/App.html` (Mac/Linux) or `(Get-Item dist/App.html).length / 1MB` (Windows)

## 4. Never edit files in `dist/`

Always edit source files in `src/`. The `dist/` folder is generated by the build step.

## 5. No routing, no lazy loading, no localStorage

Google Apps Script runs inside an iframe. These APIs are not available:
- No `window.location` or client-side routing (React Router)
- No `localStorage` or `sessionStorage`
- No dynamic `import()` or code splitting
- No `fetch()` to external URLs from the client (use GAS server functions instead)

## 6. All data goes through GAS server functions

- Client code calls `google.script.run.*` to talk to the server
- Server functions talk to Google Sheets, Drive, Gmail, etc.
- Never make direct `fetch()` calls from the React client to external APIs

## 7. State file (`.gas-app/state.json`)

- Never delete or overwrite state without reading it first
- Update incrementally: read → merge → write
- Never expose state contents to the user unless debugging

## 8. Zero manual setup for the user

- **Never hardcode a spreadsheet ID** in source code, and **never ask the user
  to create a spreadsheet**. The server creates its own database on first run
  and stores the ID in Script Properties (see the **gas-sheet-db** skill).
- Secrets and API keys go in `PropertiesService.getScriptProperties()` — never
  in source code, never in git.

## 9. One deployment, stable URL

Create ONE deployment on the first push, save its `deploymentId` in
`.gas-app/state.json`, and always redeploy with `--deploy-id` afterwards.
Creating a new deployment per push generates a new URL every time and breaks
every link the user has shared. Apps Script also caps projects at **200 saved
versions** (no API to delete) — warn the user before they hit it.

## 10. `appsscript.json` must have the `webapp` block

After `clasp create`, verify `appsscript.json` has:
```json
{
  "webapp": {
    "access": "MYSELF",
    "executeAs": "USER_ACCESSING"
  }
}
```
The `access` field is set based on user preference. `executeAs` must always be `"USER_ACCESSING"`.
